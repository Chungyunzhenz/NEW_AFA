## Context

專案自 v1.1 文件撰寫後經過數天密集開發，**資料管線基礎設施**大幅擴充：
- **Backend**: FastAPI + SQLAlchemy + SQLite，含 3 個 ML 模型 (Prophet/SARIMA/XGBoost)、排程器、多來源資料收集器
- **Frontend**: React 19.2 + Vite 8.0 + D3 7.9 + Recharts 3.8 + Zustand 5.0 + Tailwind 4.2，含互動地圖、圖表、上傳精靈、聊天機器人
- **已支援 20 種作物**，各含獨立 JSON 設定檔 (含 Prophet/SARIMA/XGBoost 超參數)
- **新增**: CODiS 歷史天氣資料管線 (18 站、2005–2025 共 327 個 CSV)、產量資料匯入器、資料品質稽核/清洗/重配對工具腳本
- **缺少**: Docker 容器化、CI/CD、測試套件、認證授權、日誌監控、通知系統、前端路由 (已安裝 react-router-dom 但未啟用)、資安合規、無障礙 (WCAG)、備份 DR

### 1.2 資料流程 (現況)

```
                            ┌── 即時資料管線 ──────────────────────────────┐
                            │                                             │
[AMIS API]  ──daily (6AM)──→│ AMISDataCollector ──→ TradingData 表        │
[CWA API]   ──daily (3AM)──→│ CWAWeatherCollector → WeatherData 表       │
                            │                                             │
                            └─────────────────────────────────────────────┘

                            ┌── 歷史資料回補管線 (NEW) ───────────────────┐
                            │                                             │
[CODiS 網站] ─download──→   │ download_codis.py → data/codis_downloads/  │
             ─import───→    │ import_codis_csv.py → WeatherData 表       │
[AMIS API]   ─backfill──→  │ backfill_trading.py → TradingData 表       │
[CWA API]    ─backfill──→  │ backfill_weather.py → WeatherData 表       │
[產量 CSV]   ─import───→   │ load_production_csv.py → ProductionData 表 │
                            │ production_collector.py (服務層)            │
                            └─────────────────────────────────────────────┘

[CSV/Excel] ──upload──→ FileUploadService → TradingData / ProductionData 表

                            ┌── 資料品質管線 (NEW) ──────────────────────┐
                            │ audit_data.py       → 品質報告             │
                            │ clean_all_trading.py → 清洗異常值          │
                            │ rematch_crop_ids.py  → 重配對 crop_id      │
                            └────────────────────────────────────────────┘

                              ↓ 清洗後資料
                    ┌─────────────────────┐
                    │  APScheduler (每週日) │
                    │  ModelTrainer        │
                    │  ├─ Prophet          │
                    │  ├─ SARIMA           │
                    │  └─ XGBoost          │
                    │  → EnsemblePredictor │
                    │  → ModelEvaluator    │
                    └─────────────────────┘
                              ↓
                  Prediction 表 + ModelRegistry 表
                              ↓
              FastAPI REST API (/api/v1, 8 個 router)
                              ↓
              React Frontend (localhost:5173)
              ├─ D3 台灣地圖 (choropleth, TopoJSON)
              ├─ Recharts 時間序列圖 (5 種圖表元件)
              ├─ 紅綠燈警示面板
              ├─ 資料上傳精靈 (5 步驟)
              └─ 聊天機器人 (規則式)
```

### 1.4 現況風險揭露

> 以下根據 2026-03-23 實際程式碼重新盤點。

#### CRITICAL 等級

| # | 風險項目 | 說明 | 影響 | v1.1 狀態 |
|---|---------|------|------|-----------|
| C1 | **SSL 驗證全域關閉** | `config.py` 預設 `VERIFY_SSL=False`，外部 API 呼叫未驗證憑證 | 中間人攻擊 (MITM) | ⬜ 未修正 |
| C2 | **無認證機制** | 所有 API endpoint 無須登入，含上傳、同步觸發、模型重訓 | 未授權存取與資料竄改 | ⬜ 未修正 |
| C3 | **Debug 模式預設開啟** | `config.py` 預設 `DEBUG=True` | 可能洩漏敏感資訊 | ⬜ 未修正 |
| C4 | **上傳檔案暫存無上限** | `file_upload_service.py` 的 `_upload_store` (dict)，30 分鐘 TTL，無容量限制 | OOM 記憶體耗盡 | ⬜ 未修正 |

#### HIGH 等級

| # | 風險項目 | 說明 | 影響 | v1.1 狀態 |
|---|---------|------|------|-----------|
| H1 | **CORS 過度寬鬆** | `main.py` 設定 `allow_methods=["*"]`, `allow_headers=["*"]` | 增加跨站攻擊面 | ⬜ 未修正 |
| H2 | **外部 API 無重試機制** | AMIS/CWA API 呼叫失敗直接回傳空陣列，無 retry/backoff | 資料缺失 | ⬜ 未修正 |
| H3 | **排程任務失敗無通知** | scheduler job 錯誤只寫 log | 延遲發現系統問題 | ⬜ 未修正 |
| H4 | **Foreign Key 可為 NULL** | `TradingData.crop_id/market_id`、`ProductionData.crop_id/county_id` 等設為 `nullable=True` | 資料完整性無保障 | ⬜ 未修正 (設計決策：允許先寫入再配對) |

#### 環境設定風險

| # | 風險項目 | 說明 | v1.1 狀態 |
|---|---------|------|-----------|
| E1 | **`.env.example` 不完整** | 僅含 `CWA_API_KEY`，缺少 `DEBUG`、`CORS_ORIGINS`、`VERIFY_SSL`、`DATABASE_URL` 等 | ⬜ 未修正 |
| E2 | **`requirements.txt` 版號寬鬆** | 全部使用 `>=` 最低版號，無 lock file | ⬜ 未修正 |
| E3 | **前端無安全標頭** | `index.html` 未設定 CSP、X-Frame-Options 等 | ⬜ 未修正 |

#### 新增發現 (v2.0)

| # | 風險項目 | 說明 | 等級 |
|---|---------|------|------|
| N1 | **CODiS 下載腳本無錯誤重試** | `download_codis.py` 單次失敗即跳過該站年份 | MEDIUM |
| N2 | **data/ 目錄未納入 .gitignore** | 327 個 CSV (數十 MB) 可能被意外 commit | MEDIUM |
| N3 | **Alembic 無任何版本** | `alembic/versions/` 為空，DB schema 變更無追蹤 | MEDIUM |

### 2.1 基礎建設 (Infrastructure)
