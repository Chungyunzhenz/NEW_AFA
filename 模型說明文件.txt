  機器學習模型說明文件
  台灣農產品產銷預測系統

一、為什麼需要預測模型？
------------------------------------------------------------

農產品價格受到多種因素影響：
  - 季節性供需（冬季甘藍盛產價格低，夏季減產價格高）
  - 天氣變化（溫度影響生長速度，降雨影響收成品質）
  - 極端事件（颱風來襲造成產量驟降、價格暴漲）
  - 市場行為（節日需求、進出口變化）

人工判斷難以同時考慮這麼多因素，因此使用機器學習模型
從歷史資料中自動學習這些模式，產出未來價格預測。


二、三個預測模型介紹
------------------------------------------------------------

【模型 1】Prophet（先知模型）
------------------------------------------------------------
開發者：Meta（Facebook）開源專案
原  理：將時間序列拆解成三個成分
        價格 = 趨勢 + 季節性 + 節假日效應

  趨勢成分 ── 長期價格是上升還是下降？
  季節性成分 ── 每年的哪幾個月價格特別高或低？
  節假日效應 ── 特定事件（如春節、颱風）造成的價格異常

擅  長：
  - 自動偵測年度季節性（例如甘藍 7-8 月必漲）
  - 對缺漏資料有容忍度
  - 可加入外部因素（天氣、颱風）作為額外迴歸變數

適用場景：
  - 季節性明顯的作物（甘藍、萵苣、西瓜）
  - 長期趨勢穩定的品項

在本系統的表現：
  - 甘藍預測中獲得 99.8% 的集成權重
  - 在「正常市場環境」下預測最為準確


【模型 2】SARIMA（季節性自回歸整合移動平均模型）
------------------------------------------------------------
全  名：Seasonal AutoRegressive Integrated Moving Average
原  理：根據過去的價格推算未來的價格

  自回歸(AR) ── 這個月的價格跟前幾個月有關
  移動平均(MA) ── 預測誤差的修正
  整合(I) ── 讓不穩定的價格序列變穩定
  季節性(S) ── 考慮每年重複的 12 個月循環

擅  長：
  - 統計學理論嚴謹，結果可解釋
  - 自動選擇最佳參數（使用 pmdarima 套件）
  - 不需要外部特徵，純粹從歷史價格推算

適用場景：
  - 價格走勢穩定、變化不劇烈的作物
  - 作為基準模型（baseline）與其他模型比較

在本系統的表現：
  - 提供穩定的基準預測
  - 在價格趨勢平穩時表現良好


【模型 3】XGBoost（極端梯度提升樹）
------------------------------------------------------------
全  名：eXtreme Gradient Boosting
原  理：建立數百棵決策樹，每棵樹修正前一棵的錯誤

  第1棵樹 ── 做出初步預測
  第2棵樹 ── 專注修正第1棵的預測誤差
  第3棵樹 ── 再修正第2棵的殘餘誤差
  ... 以此類推，200 棵樹逐步精進

擅  長：
  - 能同時考慮所有特徵（天氣、颱風、歷史價格、季節）
  - 自動判斷哪些因素最重要（特徵重要性排名）
  - 能學習非線性關係（例如「颱風+高溫+盛產季」的組合效應）

特徵輸入（共 20+ 個特徵）：
  歷史價格特徵 ── lag_1, lag_2, lag_3, lag_6, lag_12
  滾動統計 ── 3/6/12 個月的平均值和標準差
  日曆特徵 ── 月份、季度、是否為產季
  天氣特徵 ── 溫度、降雨、溫度異常
  颱風特徵 ── 是否颱風月、颱風強度、距颱風天數、
               颱風後 1-2 個月標記、極端降雨

適用場景：
  - 受天氣和颱風影響大的作物（青蔥、甘藍）
  - 需要解釋「哪些因素影響了預測」

在本系統的表現（甘藍 XGBoost 特徵重要性）：
  1. 歷史價格 ── 29.7%（最重要）
  2. 颱風後1個月 ── 11.9%（颱風來了價格會漲）
  3. 交易量 ── 10.8%
  4. 平均溫度 ── 9.5%（天氣熱產量少價格漲）
  5. 前1個月價格 ── 7.0%


三、為什麼要集成（Ensemble）？
------------------------------------------------------------

問題：三個模型各有專長，但沒有一個在所有情況都最準。

  Prophet ── 正常時期最準，但遇到突發事件反應慢
  SARIMA  ── 趨勢穩定時很好，但無法考慮天氣因素
  XGBoost ── 能捕捉颱風影響，但在平穩時期可能過度反應

解決：讓三個模型「投票」，準確度高的投票權重大。

集成方法：反向 MAPE 加權（Inverse-MAPE Weighted Ensemble）

  步驟 1：三個模型各自預測未來價格
  步驟 2：用測試集計算各模型的 MAPE（平均絕對百分比誤差）
  步驟 3：MAPE 越小 → 該模型越準 → 權重越大

  權重公式：
    weight_i = (1 / MAPE_i) / Σ(1 / MAPE_j)

  舉例（甘藍）：
    Prophet MAPE = 5%   → weight = (1/5) / (1/5+1/50+1/80) = 99.8%
    SARIMA  MAPE = 50%  → weight = (1/50) / (1/5+1/50+1/80) = 0.1%
    XGBoost MAPE = 80%  → weight = (1/80) / (1/5+1/50+1/80) = 0.1%

  集成預測值 = Prophet預測×0.998 + SARIMA預測×0.001 + XGBoost預測×0.001

好處：
  1. 降低單一模型失準的風險
  2. 自動適應 ── 不同作物會有不同的權重分配
  3. 穩健性 ── 即使某個模型完全失準，其他模型能補位


四、為什麼建議使用這三個模型？
------------------------------------------------------------

1. 互補性強
   Prophet 捕捉季節規律 + SARIMA 追蹤趨勢 + XGBoost 感知外部因素
   三者各覆蓋不同面向，合在一起最完整。

2. 適合資料規模
   我們有 170 個月度資料點。
   深度學習（LSTM、Transformer）需要數千個資料點才有效，
   用在 170 個點上會嚴重過擬合（記住資料而不是學會規律）。

3. 可解釋性
   政府單位需要知道「為什麼預測價格會上漲」。
   XGBoost 的特徵重要性排名可以回答這個問題：
   「因為上個月有颱風（排名第2），加上氣溫偏高（排名第4）」

4. 運算效率
   三個模型合計訓練時間約 5-10 分鐘（單一作物），
   不需要 GPU，一般電腦即可執行。


五、訓練規則
------------------------------------------------------------

資料分割：
  ┌─────────────────────────────────────────────┐
  │ 全部月度資料                                  │
  │                                               │
  │  ├── 最近 90 天 → 保留集（完全不碰）          │
  │  └── 剩餘資料                                 │
  │       ├── 70% → 訓練集（模型學習用）          │
  │       └── 30% → 測試集（評估準確度用）        │
  └─────────────────────────────────────────────┘

評估指標：
  MAE  ── 平均絕對誤差（預測值與實際值的平均差距）
  RMSE ── 均方根誤差（對大誤差更敏感）
  MAPE ── 平均絕對百分比誤差（用百分比衡量，直覺好懂）

自動重訓：
  系統排程每週日凌晨 02:00 自動重新訓練模型，
  確保模型持續學習最新的市場資料。
