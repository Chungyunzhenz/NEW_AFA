# 農產品價格預測模型完整報告

## 1. 報告目的

本報告針對已整理完成的每日時間序列資料，建立並比較多種價格預測模型，用於評估各作物未來價格走勢的預測能力。

本階段只處理「預測模型」任務；回歸分析資料已另行整理，可供其他應用程式使用。

## 2. 資料來源與處理方式

資料來源為：

```text
AFA-other/model_ready/forecast/
```

每個作物皆已整理為每日時間序列資料，主要欄位如下：

| 欄位 | 說明 |
| --- | --- |
| `ds` | 日期 |
| `y` | 每日全國加權平均價格 |
| `price_avg_weighted` | 以交易量加權後的平均價格 |
| `volume_total` | 當日總交易量 |
| `ln_volume_total` | 總交易量取 log 後的值 |
| `price_lag_*` | 過去價格落後特徵 |
| `price_roll_mean_*` | 過去價格移動平均 |
| `price_roll_std_7` | 7 日價格波動程度 |
| `month_sin`, `month_cos` | 月份季節性週期特徵 |

價格採用交易量加權平均：

```text
price_avg_weighted = sum(平均價 * 交易量) / sum(交易量)
```

資料切分採時間順序，不進行隨機切分，以避免未來資訊洩漏。模型使用訓練期間資料訓練，再於測試期間評估。

## 4. 使用模型

本次比較以下模型：

| 模型 | 類型 | 說明 |
| --- | --- | --- |
| `naive_lag1` | Baseline | 使用前一日價格作為預測 |
| `seasonal_naive_lag7` | Baseline | 使用前 7 日價格作為預測 |

## 5. 回歸式價格預測指標

價格預測本質上是連續數值預測，因此主要使用以下指標：

| 指標 | 說明 |
| --- | --- |
| MAE | 平均絕對誤差，越低越好 |
| RMSE | 均方根誤差，越低越好，對大誤差較敏感 |
| MAPE | 平均絕對百分比誤差，越低越好 |
| R2 | 解釋能力，越接近 1 越好 |

## 6. 各作物最佳預測模型

依 RMSE 選擇最佳模型，結果如下：

| 作物 | 最佳模型 | MAE | RMSE | MAPE | R2 |
| --- | --- | ---: | ---: | ---: | ---: |
| 甘藍 `cabbage` | XGBoost | 1.4158 | 2.1090 | 7.5252% | 0.9538 |
| 青江菜 `bok_choy` | XGBoost | 2.2629 | 3.9583 | 8.3266% | 0.9261 |
| 花椰菜 `cauliflower` | XGBoost | 3.4573 | 5.7756 | 8.9067% | 0.8498 |
| 青蔥 `green_onion` | Ridge | 6.1748 | 10.7128 | 8.3430% | 0.9565 |
| 萵苣 `lettuce` | Ridge | 3.8201 | 5.4444 | 11.2568% | 0.9113 |

整體來看，XGBoost 在甘藍、青江菜、花椰菜上表現最佳；Ridge 在青蔥與萵苣上以 RMSE 作為標準時表現最佳。

## 8. 預警分類評估

除了連續價格預測，本報告也將預測結果轉換成預警分類任務，以評估模型是否能偵測價格上漲或大幅上漲。

分類任務如下：

| 任務 | 定義 |
| --- | --- |
| `direction_up` | 價格是否上漲 |
| `rise_ge_5pct` | 價格是否上漲超過 5% |
| `rise_ge_10pct` | 價格是否上漲超過 10% |

分類評估指標如下：

| 指標 | 說明 |
| --- | --- |
| Accuracy | 整體分類正確率 |
| Precision | 模型預測為預警時，有多少是真的預警 |
| Recall | 實際發生預警時，模型抓到多少 |
| F1-score | Precision 與 Recall 的平衡 |

對預警任務而言，Accuracy 不應單獨使用，因為大幅上漲事件通常較少，模型即使都預測不發生，也可能有偏高的 Accuracy。建議優先看 Recall 與 F1-score。

## 9. 大幅上漲預警表現

以下列出「上漲超過 10%」任務中，各作物 F1-score 最佳模型：

| 作物 | 最佳模型 | Accuracy | Precision | Recall | F1-score |
| --- | --- | ---: | ---: | ---: | ---: |
| 甘藍 | XGBoost | 0.8597 | 0.6218 | 0.3008 | 0.4055 |
| 青江菜 | LightGBM | 0.8113 | 0.6807 | 0.2425 | 0.3576 |
| 花椰菜 | LightGBM | 0.8087 | 0.7457 | 0.3386 | 0.4657 |
| 青蔥 | LightGBM | 0.8216 | 0.4613 | 0.5019 | 0.4807 |
| 萵苣 | LightGBM | 0.8221 | 0.6320 | 0.4540 | 0.5284 |

整體而言，LightGBM 在預警分類任務上表現較穩定，尤其是青蔥與萵苣的大幅上漲預警，其 Recall 與 F1-score 都相對較好。

## 10. 結論

本次結果顯示：

1. 價格數值預測方面，XGBoost 在甘藍、青江菜、花椰菜上表現最佳。
2. 青蔥與萵苣若以 RMSE 作為主要標準，Ridge 表現最佳。
3. 若以預警分類為目標，LightGBM 在多數作物上有較佳的 F1-score。
4. SARIMA 在目前設定下表現不佳，可能不適合直接套用在每日價格資料，或需要針對不同作物重新調參。
5. Prophet 在甘藍上成功執行，但表現不佳；其他作物出現 backend 相容問題，暫不建議作為主要模型。

## 12. 後續改善方向

後續若要提升預測與預警能力，建議加入以下資料：

1. 氣象資料：溫度、雨量、颱風、極端降雨。
2. 產量資料：各縣市種植面積、收穫量、產期。
3. 節慶與週期：農曆春節、中元節、中秋節等需求波動。
4. 市場分區資料：不同批發市場可能有不同價格傳導效果。
5. 多步預測設定：分別建立 1 日、5 日、7 日、14 日預測模型。
